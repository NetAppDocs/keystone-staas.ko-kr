= 키스톤의 성능 서비스 수준
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Keystone STaaS는 각 스토리지 유형별로 사전 정의된 성능 서비스 수준(PSL)을 제공합니다. 각 PSL에는 성능 목표(IOPS, 처리량 및 지연 시간), 지원 프로토콜, 플랫폼 및 최소 용량 요구 사항이 명시되어 있습니다. 사용 가능한 PSL을 검토하여 워크로드에 적합한 서비스 수준을 선택하고 구독을 계획하십시오.

Keystone 구독에는 하나 이상의 스토리지 유형에 걸쳐 여러 개의 PSL이 포함될 수 있습니다. 각 PSL에는 약정 용량이 있으며, 이는 구독 기간 동안 해당 서비스 수준에 대해 청구되는 최소 용량입니다.


NOTE: AFX 구독은 AFX 스토리지 유형으로 제한되며 현재 하나의 PSL인 Extreme만 제공합니다.



== 성능 서비스 수준 인스턴스

성능 서비스 레벨(PSL)은 환경 내에서 성능 서비스 레벨 인스턴스(PSLI)라고 하는 여러 인스턴스를 가질 수 있습니다. 각 PSLI는 해당 성능 서비스 레벨에 할당된 별도의 스토리지 시스템이며, 약정 용량은 PSLI별로 측정됩니다.

* 통합 스토리지 또는 블록 최적화 스토리지의 경우, PSLI는 내부 또는 외부에 연결된 스토리지 미디어를 갖춘 HA 쌍의 2개 스토리지 컨트롤러로 구성된 스토리지 어레이입니다.
* AFX 스토리지에서 PSLI는 단일 AFX 스토리지 컨트롤러입니다. AFX는 컨트롤러(컴퓨팅)와 드라이브 셸프(스토리지)가 독립적이며 개별적으로 확장할 수 있는 분산형 아키텍처를 사용합니다. IOPS 및 처리량 목표는 컨트롤러별로 적용되며, 약정 용량은 클러스터 내의 모든 컨트롤러와 드라이브 셸프를 포함하는 AFX 클러스터 수준에서 측정됩니다.


IOPS, 처리량(GBps), 지연 시간(ms)과 같은 성능 지표는 PSLI 단위로 측정됩니다.


TIP: NetApp Keystone STaaS 성능 서비스 수준에 대한 자세한 서비스 설명을 사용할 수 있습니다.  https://www.netapp.com/services/keystone/terms-and-conditions/["여기"^] .

지원되는 데이터 스토리지 유형(통합, 블록 최적화, AFX, 오브젝트 및 클라우드 서비스)에 대한 기본 성능 서비스 수준은 다음 섹션에 설명되어 있습니다.

== 통합형, 블록 최적화형 및 AFX 스토리지에 대한 성능 측정 조건

기본 성능 서비스 수준 측정항목은 다음 조건에 따라 달라집니다.

* 성능 서비스 수준은 ONTAP 9.8 이상을 지원합니다.
* 통합 스토리지의 경우
+
** *IOPS*: NFS를 탑재한 ONTAP 9.16.1의 경우, 각 성능 수준 인스턴스는 70% 읽기, 30% 쓰기 비율, 8KB 블록 크기, 1ms 대기 시간(표준의 경우 4ms)의 랜덤 액세스를 지원합니다.
** *처리량*: NFS를 탑재한 ONTAP 9.16.1의 경우, 각 성능 수준 인스턴스는 100% 읽기와 32KB 블록 크기의 순차적 액세스를 지원합니다.


* 블록 최적화된 스토리지의 경우
+
** *IOPS*: FCP가 포함된 ONTAP 9.16.1의 경우, 각 성능 수준 인스턴스는 70% 읽기, 30% 쓰기 비율, 8KB 블록 크기, 1ms 지연 시간의 랜덤 액세스를 지원합니다.
** *처리량*: FCP가 포함된 ONTAP 9.16.1의 경우 각 성능 수준 인스턴스는 100% 읽기와 64KB 블록 크기의 순차적 액세스를 지원합니다.


* AFX 스토리지의 경우,
+
** *IOPS*: NFSv4를 사용하는 ONTAP 9.18.1의 경우, 각 성능 레벨 인스턴스는 70% 읽기 및 30% 쓰기 비율과 4KB 블록 크기로 임의 액세스를 지원합니다.
** *처리량*: NFSv4를 사용하는 ONTAP 9.18.1의 경우, 각 성능 레벨 인스턴스는 100% 읽기 및 64KB 블록 크기로 순차 액세스를 지원합니다.


* 지연 시간은 다음 사항을 포함하지 않습니다.
+
** 애플리케이션 또는 호스트 지연 시간
** 컨트롤러 포트 간 또는 컨트롤러 포트로 전송되는 고객의 네트워크 지연 시간
** FabricPool의 경우 데이터 전송과 관련된 오버헤드를 오브젝트 저장소로 이동합니다


* 지연 시간 값은 MetroCluster 쓰기 작업에 적용할 수 없습니다. 이러한 쓰기 작업은 원격 시스템의 거리에 따라 달라집니다.
* _expected IOPS_는 계층화 정책이 "없음"으로 설정되고 클라우드에 블록이 없는 경우에만 FabricPool를 대상으로 합니다. _expected IOPS_는 SnapMirror 동기식 관계가 아닌 볼륨을 대상으로 합니다.
